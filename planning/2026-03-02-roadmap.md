# Daily Review Roadmap

Goal: A stranger can hit the URL, sign up, understand the app, generate cards, and review them daily without rough edges.

## Phase 1 — Core Improvements

**Card data model rework**
- Replace hidden `context` field with a visible, editable `back` field
- Shown to users after they answer (so they can learn from feedback)
- Also fed to the LLM grader as reference material
- Users can edit both `front` and `back` on any card

**UX completeness**
- New-card triage: browse/cycle through proposed cards before committing, with accept or discard actions per card
- Self-grade option (alternative to AI grading)
- Card search and filtering
## Phase 2 — Onboarding & Retention

**Guided onboarding for new users**
- First-time user tutorial explaining the core loop (generate → triage → review)
- Seed cards or interactive walkthrough so users experience review immediately

**Email notifications**
- Daily reminder when cards are due
- User preferences (frequency, opt-out)

## Phase 3 — Visual Polish

- Animations and transitions (card flips, page transitions, loading states)
- Empty states, error states, loading skeletons
- Mobile responsiveness
- Consistent visual design pass

## Phase 4 — Engineering Quality

- Tests (currently zero)
- CI/CD pipeline (GitHub Actions: test on push, deploy on merge)
- Error handling across the stack
- Third-party auth providers (GitHub/Google via BetterAuth)
- Polish the `/flashcards` generation skill
